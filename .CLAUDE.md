# Vision Extraction Benchmark — CLAUDE.md

쇼핑 스크린샷에서 상품 정보를 추출하는 Vision 모델 벤치마크 프로젝트.
Gemini / GPT-4o 두 모델의 추출 퀄리티를 동일한 조건에서 비교한다.

---

## 디렉토리 구조

```
vision-test/
  images/                          # 테스트용 쇼핑 스크린샷 원본
    musinsa_pants_001/             # 상품당 폴더로 관리
      01.png                       # 단일 이미지면 01.png 하나만
      02.png                       # 같은 상품 추가 정보...
    zigzag_skirt_003/
      01.png
    ...

  manifests/
    dataset.jsonl                  # 이미지 목록 인덱스

  prompts/
    extraction_prompt_gemini.txt   # Gemini용 추출 프롬프트
    extraction_prompt_gpt.txt      # GPT용 추출 프롬프트

  results/
    gemini/                        # Gemini 추출 결과
      musinsa_pants_001_01.jsonl
      musinsa_pants_001_02.jsonl
      zigzag_skirt_003_01.jsonl
      ...
    gpt/                           # GPT 추출 결과
      musinsa_pants_001_01.jsonl
      ...
```

---

## 파일 네이밍 규칙

### 이미지
- `images/{상품ID}/{번호}.png`
- 상품ID는 플랫폼_카테고리_번호 권장 (예: `musinsa_pants_001`, `zigzag_skirt_003`)
- 같은 상품의 여러 이미지는 같은 폴더에 01.png, 02.png로 추가
- 단일 이미지면 01.png 하나만

### 결과 파일
- `results/{모델명}/{파일명}_{실행번호}.jsonl`
- 실행번호는 01부터 두 자리 (01, 02, 03...)
- 같은 이미지를 여러 번 돌릴 때 실행번호로 구분
- 예: `results/gemini/musinsa_pants_001_01.jsonl`, `results/gemini/musinsa_pants_001_02.jsonl`

---

## dataset.jsonl 구조

한 줄 = 상품 하나. JSON Lines 포맷.
`images`는 배열로 관리 — 단일 이미지면 요소 하나, 다중이면 여러 개.

```jsonl
{"id": "musinsa_pants_001", "images": ["images/musinsa_pants_001/01.png", "images/musinsa_pants_001/02.png"]}
{"id": "zigzag_skirt_003", "images": ["images/zigzag_skirt_003/01.png"]}
```

## 추출 결과 스키마

추출되는 상품 정보의 필드 정의:

```json
{
  "product_name": "오버핏 코튼 후드집업",
  "original_price": 39000,
  "is_discounted": true,
  "discounted_price": 27300,
  "discount_rate": 30,
  "review_count": 1284,
  "review_score": 4.8,
  "category": "후드티",
  "color": "오프화이트",
  "fit": "오버핏",
  "style_keywords": ["캐주얼", "스트릿"],
  "shot_type": "모델착용샷",
  "visibility": "양호",
  "wishlist_count": 3200,
  "delivery_info": "내일 도착",
  "brand_name": "무신사 스탠다드"
}
```

### 필드 타입 및 허용값

| 필드 | 타입 | 없을 때 | 비고 |
|------|------|---------|------|
| product_name | string | null | |
| original_price | number | null | 숫자만, 단위 제거 |
| is_discounted | boolean | false | |
| discounted_price | number | null | |
| discount_rate | number | null | 숫자만 (% 제거) |
| review_count | number | null | |
| review_score | number | null | |
| category | string | null | 자유 텍스트, 모델이 자유롭게 추출 |
| color | string | null | |
| fit | string | null | 자유 텍스트, 모델이 자유롭게 추출 |
| style_keywords | array of string | [] | 자유 텍스트, 모델이 자유롭게 추출 |
| shot_type | string (열거형) | null | 아래 허용값 참고 |
| visibility | string (열거형) | null | 아래 허용값 참고 |
| wishlist_count | number | null | |
| delivery_info | string | null | |
| brand_name | string | null | |

### 열거형 허용값

- **shot_type**: 모델착용샷 / 흰배경단독샷 / 행거샷 / 기타
- **visibility**: 양호 / 부분가림 / 불량

## 결과 JSONL 구조

한 번 실행 = 한 줄. 메타데이터 + 추출 결과를 함께 저장한다.
0과 null을 구분해서 저장한다.

```jsonl
{"run": 1, "model": "gemini", "image_id": "hello", "timestamp": "2026-01-28T14:30:00Z", "result": { ...추출된 JSON... }}
```

---

## 프롬프트 파일

모델별로 프롬프트를 따로 관리한다.
`prompts/extraction_prompt_gemini.txt`
`prompts/extraction_prompt_gpt.txt`

기본 지시 구조는 동일하게 유지하되, 모델 특성에 맞게 튜닝한다.
프롬프트 변경 시 결과 파일의 실행번호를 올려서 이전 결과와 구분한다.

---

## 주의사항

- 결과 파일은 덮어쓰지 않는다. 재실행 시 항상 실행번호를 올린다.
- style_keywords는 열거형이 아니므로 정답 비교 시 완전 일치가 아닌 유사도 기준으로 평가한다.
