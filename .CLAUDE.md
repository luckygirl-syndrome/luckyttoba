# Vision Extraction Test — 기술 문서

쇼핑 스크린샷 이미지에서 Vision LLM이 상품 정보를 추출하는 성능을 평가하는 실험 프로젝트.

---

## 프로젝트 개요

### 목적
- Gemini, GPT-4o의 Vision 모델이 쇼핑 스크린샷에서 얼마나 정확하게 상품 정보를 추출하는지 측정
- 이후 앱 개발에서 사용할 모델과 프롬프트 최적화를 위한 기초 데이터 수집

### 구조
```
images/                    # 테스트 이미지
manifests/dataset.jsonl   # 이미지 목록 정의
prompts/                  # 모델별 프롬프트
results/                  # 추출 결과
run_extraction.py         # 메인 스크립트
```

---

## dataset.jsonl 포맷

JSON Lines 형식. 한 줄 = 한 테스트 케이스.

```jsonl
{"id": "product_001", "images": ["images/product_001/01.png", "images/product_001/02.png"]}
{"id": "product_002", "images": ["images/product_002/01.png"]}
```

필드:
- **id**: 상품 고유 ID
- **images**: 이미지 경로 배열 (다중 이미지 지원)

## 추출 결과 포맷

### 저장 경로
`results/{모델명}/{image_id}_{run_번호}.jsonl`

예: `results/gemini/product_001_01.jsonl`

### 파일 내용
```jsonl
{"run": 1, "model": "gemini", "image_id": "product_001", "timestamp": "2026-04-14T10:30:00Z", "result": {...}}
```

메타데이터:
- **run**: 실행 번호 (같은 이미지를 여러 번 실행할 때 증가)
- **model**: 사용 모델 ("gemini" 또는 "gpt")
- **image_id**: dataset.jsonl의 id 값
- **timestamp**: 실행 시간 (UTC ISO 8601)
- **result**: 추출된 상품 정보

### 추출된 필드 (result)

```json
{
  "product_name": "오버핏 코튼 후드집업",
  "original_price": 39000,
  "is_discounted": true,
  "discounted_price": 27300,
  "discount_rate": 30,
  "review_count": 1284,
  "review_score": 4.8,
  "category": "후드티",
  "color": "오프화이트",
  "fit": "오버핏",
  "style_keywords": ["캐주얼", "스트릿"],
  "shot_type": "모델착용샷",
  "visibility": "양호",
  "wishlist_count": 3200,
  "delivery_info": "빠른출발",
  "brand_name": "무신사 스탠다드"
}
```

필드 타입:
- `string`: 텍스트 (null 가능)
- `number`: 숫자 (null 가능)
- `boolean`: true/false
- `array`: 문자열 배열

특이사항:
- **is_discounted**: 할인 여부 (기본값 false)
- **style_keywords**: 시각적 추론 기반 (기본값 [])
- **shot_type, visibility**: 고정 값만 허용
- **0 vs null**: 0은 실제 값, null은 정보 없음

## 프롬프트 설계

### 위치
- `prompts/extraction_prompt_gemini.txt`
- `prompts/extraction_prompt_gpt.txt`

### 핵심 지침
1. **추론 vs 명시**: category, color, fit, style_keywords는 이미지에서 시각적으로 추론
2. **정확성**: review_score, original_price 등은 명시된 값만 추출
3. **포맷**: JSON Lines 응답, null/0 구분, 단위 제거
4. **허용값 제한**: shot_type, visibility, style_keywords는 정해진 값만 사용

### 프롬프트 버전 관리
프롬프트 수정 후 재실행 시 run 번호를 증가시켜 이전 결과와 구분함.

---

## 실행 흐름

1. `run_extraction.py` 실행
2. dataset.jsonl 로드
3. 각 항목에 대해:
   - 이미지 로드 (base64 인코딩)
   - 모델 API 호출 (프롬프트 + 이미지)
   - JSON 응답 파싱
   - 결과 저장 (`results/{모델}/...`)

### 다중 이미지 처리
- **Gemini**: 여러 이미지를 한 번에 전송 가능
- **GPT-4o**: 현재는 첫 번째 이미지만 처리 (향후 확장 가능)

---

## 주의사항

- **재실행**: 같은 이미지를 다시 실행하면 run 번호가 자동으로 증가 (기존 결과 유지)
- **API 비용**: Gemini와 GPT-4o 모두 Vision API 요청에 비용 발생
- **이미지 형식**: PNG, JPG, GIF, WebP 지원
- **에러 처리**: API 오류 시 해당 항목만 실패하고 다음으로 진행
