## 3. 인사이트

1. **wishlist_count 51%**: v2에서도 같은 양상 (58%). 화면에 안 보이는 케이스가 많거나, 추출 모델이 자주 놓침. 지그재그는 좋아요 기능 자체가 없음(NaN).
2. **fit 89%**: 시각 추론으로 채우기 → v2에서도 79%로 비슷. 인간 라벨러도 어려운 필드.
3. **review_score 81%**: 화면에 평점이 안 보이는 상품이 19% — 필드 자체가 결측 많음.
4. **delivery_info 65%**: 추출 정의("배송 방식")과 GT 정의("일정")가 어긋날 가능성 — v2 평가에서 실제로 그런 미스매치 확인됨.
5. **마케팅 트리거 부재**: trend_hype/bundle/confidence 측정 안 함 → 시험 2에서 별도 LLM으로 처리하려다 v2에서 통합.
