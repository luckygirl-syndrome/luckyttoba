# 시험 2: 마케팅 트리거 분류

## 목적

title_marketing_score의 입력값(trend_hype, bundle, confidence)을
상품명에서 정확히 분류하는지 검증. Impulse Score 가중치의 20%.

## 추출 대상

| 시그널 | 값 | 예시 |
|--------|-----|------|
| trend_hype | 0/1 + 해당 문구 | "랭킹1위", "인기상품" |
| bundle | 0/1 + 해당 문구 | "1+1", "세트특가" |
| confidence | 0/1 + 해당 문구 | "디자이너 추천", "리얼후기" |

## GT 데이터셋

- 상품명 최소 200개 (팀원 각 50개씩 수동 라벨링)
- 경계 케이스 합의 필수: "인기상품"→trend_hype? "한정수량"→bundle? "디자이너 추천"→confidence?

## 측정 지표

- 축별 Precision, Recall, F1
- **Recall 중요** — 시그널을 못 잡으면 Impulse Score 과소평가

## 비교 실험

| 파이프라인 | 설명 |
|-----------|------|
| (a) RoBERTa 임베딩 | 기존 정현이 파이프라인 |
| (b) Gemini Flash-Lite 프롬프트 | Vision 추출 시 프롬프트에 포함 |

같은 GT로 둘 다 돌려서 성능 비교. (b) ≥ (a)이면 (a) 드롭.

## 추가 시험 (v2)

- 0/1 이진 → 0.0~1.0 연속값 ("랭킹1위" vs "인기상품" 강도 차이)
- GT 합의가 어렵기 때문에 v2에서 고려
