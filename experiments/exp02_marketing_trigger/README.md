title_marketing_score의 입력값(trend_hype, bundle, confidence)을

## 추출 대상

| 시그널 | 값 | 예시 |
|--------|-----|------|
| trend_hype | 0/1 + 소수점 스코어 + 해당 문구 | "주문폭주", "재입고", "1위" |
| bundle | 0/1 + 소수점 스코어 + 해당 문구 | "1+1", "특가", "블프" |
| confidence | 0/1 + 소수점 스코어 + 해당 문구 | "MD소장", "실물보장", "사장pick" |

## 파이프라인

```
[01] 200개 상품명 전략적 샘플링 → GT 템플릿
  ↓
[02] e5 임베딩 → 0/1 이진 분류
  ↓
[03] LLM (Gemini) → 0/1 이진 + 소수점 스코어 (각각 독립 추출)
  ↓
[04] GT 대비 e5 vs LLM 비교 평가
  ↓
[05] 비교 시각화 + 불일치 분석
```

## 비교 실험

| 방법 | 출력 | 설명 |
|------|------|------|
| e5 임베딩 | 0/1 이진 | 축별 정의문과의 cosine similarity → threshold 기반 |
| Gemini Flash-Lite | 0/1 이진 + 소수점 | 이진값과 스코어를 각각 독립적으로 추출 |

LLM의 소수점 스코어(0.0~1.0)는 키워드 개수 + 끌림 정도를 종합한 마케팅 자극 강도.
이진값과 소수점은 서로 파생이 아닌 독립 판단 → 불일치 분석 가능
(예: score ≥ 0.5인데 binary=0인 케이스).

소수점 스코어는 아직 확정 아님 — 결과 보고 판단.

## GT 데이터셋

- 상품명 200개 (팜팜이 100 + 경현 100, 20개 중복)
- 전략적 샘플링: 양성 60 + 음성 60 + 경계 80
- 경계 케이스 합의 필수 → `configs/edge_cases.yaml`

## 측정 지표

- 축별 Precision, Recall, F1
- **Recall 중요** — 시그널을 못 잡으면 Impulse Score 과소평가
- LLM 이진 ↔ 소수점 불일치 분석
