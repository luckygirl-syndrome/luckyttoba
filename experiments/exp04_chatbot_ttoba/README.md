# 시험 4: 최종 챗봇 (또바) — 대화 품질 검증

## 목적

system_prompt(캐릭터 + SBTI + 프로필) + context_sentences(상품 + 점수 맥락)를
받아서 유저와 대화하는 또바의 맥락 이해도, 유형별 차이, 톤을 검증한다.

## 시험 항목

### a) 맥락 이해도 테스트
- 시나리오 5~10개 (유저 유형 + 상품 + 점수 조합이 다른 것)
- 각 시나리오에서 동일한 유저 발화 3개:
  1. "이거 사도 돼?" (구매 확인형)
  2. "이거 왜 이렇게 비싸?" (가격 불만형)
  3. "비슷한 거 이미 있는데..." (중복 의심형)

**체크 포인트:**
- D형 → 감정적 자극 짚어주는가?
- N형 → 필요성 물어보는가?
- E형 → 할인 자극 환기하는가?
- M형 → "대중적 = 매력 감소" 건드리는가?
- Impulse↑ → 브레이크 톤?
- Match↓ → 비추 방향?

### b) SBTI 유형 간 차이 확인
- 같은 상품, 다른 유형: DUTE vs NIMO 응답 비교
- 톤, 짚는 포인트, 추천/비추천 방향이 확실히 달라야 함

### c) 톤 & 말투 준수
- 반말 유지
- "친근하지만 날카롭게" — 불편한 지적 OK, 깎아내리기 NO
- 한 턴 3~5문장 (모바일 챗봇)

### d) context_sentences 활용도
- context에 넣은 정보를 실제로 참조하는지
- 반례: "SNS에서 수동 노출" 맥락인데 "직접 찾아봤으니 관심 확실" 응답 → 맥락 무시
